# 发现与决策

## 需求

- 做一个纯前端网页版小工具。
- 输入文本后生成适合发布小红书的图片。
- 支持长图输出。
- 支持按固定长度裁切成多张图片。
- 保持文本排版美观。
- 保持原来输入的格式。
- 需要输出项目设计文档，方便后续跟踪。
- 增加标题输入，输出图片以标题命名。
- 增加固定保存目录，导出时自动保存到设置目录并记住路径。
- 如果重名，自动使用 `标题(1).png`、`标题(1)-01.png` 等别名，不覆盖已有文件。

## 研究发现

- 当前项目目录已有 `index.html`。
- `index.html` 已包含基础控件：文本输入、输出模式、模板、字体、宽度、分段高度、字号、行距、边距、下载按钮、预览区。
- 已补齐 `styles.css` 和 `app.js`。
- 由于目标是纯前端工具，最小可行方案不需要构建系统或后端。
- 浏览器验证显示：默认长图模式生成 1 张 1080x1440 Canvas；固定分段模式可生成多张 1080x1440 Canvas。
- 下载按钮会生成 Blob URL，并触发符合命名规则的 PNG 下载文件名。
- Chromium 系浏览器可使用 File System Access API 选择保存目录；其他浏览器回退到普通下载。
- 目录选择需要在 Chrome/Edge 顶层页面中由用户点击触发；嵌入式预览窗口可能被浏览器安全策略拦截。

## 技术决策

| 决策 | 理由 |
|------|------|
| 使用原生 HTML/CSS/JS | 降低复杂度，符合纯前端和轻量工具定位 |
| 使用 Canvas 输出 PNG | 浏览器原生支持，适合精确控制图片尺寸、背景、文字和下载 |
| 使用手写文本布局逻辑 | 需要保留输入格式并避免分段切断文字行 |
| 先逐张下载分段图片 | 避免引入 ZIP 依赖，保持架构简单 |
| 文档化验收清单 | 方便后续继续开发时逐项验证 |
| 使用 IndexedDB 保存目录句柄 | 纯前端记住目录，不需要后端 |
| 使用 `(n)` 别名防覆盖 | 符合用户要求，文件名可读且不覆盖已有文件 |

## 遇到的问题

| 问题 | 解决方案 |
|------|---------|
| 技能文件读取可能受沙箱限制 | 对必要读取使用获批命令，项目文件仍在工作目录内编辑 |
| 项目尚未完整实现 | 已补齐样式、脚本、渲染、导出和验证 |
| `agent-browser` 默认启动失败或 stdin 不工作 | 改用 `agent-browser.cmd`、独立 session 和 batch JSON |
