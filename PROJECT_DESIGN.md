# 小红书图文生成器 - 项目设计文档

## 1. 项目目标

做一个纯前端网页小工具，把用户输入的文本转换为适合发布到小红书的图片。工具需要保留原始文本格式，提供美观排版，并支持两种输出方式：

- 长图输出：把全部文本生成一张连续长图。
- 固定长度裁切输出：按指定高度切成多张图片，方便作为小红书多图发布。

## 2. 使用场景

- 用户已经写好文案，希望快速生成小红书图文。
- 用户需要保留换行、空行、列表、缩进等输入格式。
- 用户需要一键导出长图，或导出多张固定高度图片。
- 用户希望不安装软件、不依赖后端，直接打开网页即可使用。

## 3. 项目范围

### 本期必须支持

| 模块 | 需求 | 验收标准 |
|------|------|----------|
| 文本输入 | 输入或粘贴正文 | 文本框可编辑，支持多行、空行、符号和缩进 |
| 标题输入 | 输入图片标题 | 标题显示在导出图片中，并作为文件名基础 |
| 格式保留 | 保留原始换行、段落、列表和缩进 | 预览和导出图片与输入结构一致 |
| 排版美化 | 自动控制字号、行距、边距、段落间距 | 默认效果适合小红书阅读，不拥挤、不松散 |
| 实时预览 | 输入和配置变化后更新预览 | 预览能反映最终图片效果 |
| 长图导出 | 生成一张完整 PNG | 点击下载后得到包含全部内容的长图 |
| 固定分段导出 | 按固定高度切图 | 可导出多张等宽、固定高度 PNG |
| 固定保存目录 | 选择并记住导出目录 | 支持的浏览器中导出时自动保存到上次选择的目录 |
| 防覆盖命名 | 重名时自动加别名 | 不覆盖已有文件，使用 `标题(1).png` / `标题(1)-01.png` |
| 输出配置 | 可选模板、字体、宽度、字号、行距、边距、分段高度 | 控件变化会影响预览和导出 |
| 纯前端 | 不需要后端、不需要复杂架构 | 静态文件可直接运行 |

### 暂不做

- 登录、云端保存、账号系统。
- AI 改写文案。
- 后端图片生成服务。
- 复杂海报模板市场。
- 移动端小程序封装。

## 4. 用户流程

1. 打开页面。
2. 在左侧输入或粘贴文案。
3. 选择模板、字体、输出宽度、字号、行距、边距。
4. 选择输出模式：
   - 长图：预览一张完整长图。
   - 固定分段：预览多张图片。
5. 点击下载：
   - 长图模式下载一张 PNG。
   - 分段模式下载所有分段 PNG。

## 5. 信息架构

页面采用左右工作台布局：

| 区域 | 内容 |
|------|------|
| 顶部栏 | 工具名称、当前状态 |
| 左侧编辑区 | 文本输入、示例、清空、输出模式、排版配置、下载按钮 |
| 右侧预览区 | 根据当前配置生成图片预览，展示图片张数 |

移动端可调整为上下布局：编辑区在前，预览区在后。

## 6. 视觉与排版原则

视觉目标：干净、安静、有纸感，重点服务长文本阅读，而不是做成营销落地页。

排版原则：

- 默认画布宽度优先使用 1080px，符合小红书常见图片尺寸。
- 正文使用较大的中文阅读字号，默认 46px。
- 默认行距 1.75，保证截图后仍清晰。
- 边距默认 118px，避免文字贴边。
- 段落间距应略大于行距，让长文更容易扫读。
- 保留空行，避免把用户刻意安排的节奏压平。
- 长词、英文、链接需要自动换行，不能溢出画布。

推荐渲染路径：

```text
输入文本
  -> 读取配置
  -> 按换行拆成段落
  -> 保留空行和缩进
  -> Canvas 文本测量与自动换行
  -> 生成排版行列表
  -> 根据输出模式绘制为一个或多个 Canvas
  -> 预览 Canvas
  -> 下载 PNG
```

## 8. 核心实现细节

### 8.1 保留输入格式

需要保留：

- 换行。
- 空行。
- 列表符号，例如 `-`、`*`、`1.`、`01.`。
- 中文和英文混排。
- 行首缩进。
- 用户输入的段落顺序。

实现建议：

- 不对用户文本做 `trim()` 后再整体渲染，避免删除用户故意输入的前后空行。
- 使用 `text.split(/\r?\n/)` 逐行处理。
- 空行作为独立垂直间距处理。
- 每一行内部使用 Canvas `measureText` 计算换行。
- 对缩进行保留前导空格的视觉宽度。

### 8.2 长图输出

长图模式先计算全部文本高度，再创建对应高度的 Canvas。

约束：

- 最小高度不低于一个基础画面高度，避免内容少时图片太矮。
- 最大高度需要注意浏览器 Canvas 限制；如果文本极长，应提示用户改用固定分段模式。

### 8.3 固定分段输出

分段模式按配置的分段高度生成多张 Canvas。

约束：

- 每张图宽度一致。
- 每张图高度为用户选择的固定高度。
- 尽量避免一行文字被切成上下两半。
- 分段图片文件名带序号，例如 `xiaohongshu-page-01.png`。

### 8.4 下载策略

纯前端下载可以用：

```text
canvas.toBlob()
  -> URL.createObjectURL(blob)
  -> 创建 a 标签并触发 download
  -> URL.revokeObjectURL()
```

多图下载逐张触发浏览器下载。若后续需要更稳，可以再引入 ZIP，但本期先不增加依赖。

### 8.5 标题、目录与防覆盖命名

- 标题输入同时用于图片内标题和导出文件名。
- 文件名中的非法字符会被替换，空标题回退为 `xiaohongshu`。
- 支持 File System Access API 的浏览器可选择固定保存目录，并通过 IndexedDB 记住目录句柄。
- 导出前检查目录中是否存在同名文件，存在则自动递增别名：
  - 长图：`标题.png`、`标题(1).png`、`标题(2).png`
  - 分段：`标题-01.png`、`标题-02.png`
  - 分段重名：`标题(1)-01.png`、`标题(1)-02.png`
- 如果浏览器不支持固定目录或权限失效，回退到普通下载。
